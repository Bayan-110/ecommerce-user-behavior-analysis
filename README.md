# 电商用户行为分析项目

> 2200万用户 · 2.25亿条真实数据 · 转化漏斗 · 用户分层 · 流失预测 · AB测试设计

## 📌 项目背景

本项目基于 **Synerise 公开数据集**（2025年发布），对某电商平台2200万用户的2.25亿条行为记录进行全面分析。

**核心业务问题**：
1. 为什么加购后转化率只有39%？如何提升？
2. 哪些用户最可能流失？如何精准召回？

## 💡 核心发现

### 发现一：加购后转化率低，但原因不是“遗忘”

| 指标 | 数值 |
|------|------|
| 浏览→加购转化率 | 10.6% |
| 加购→购买转化率 | 39.0% |
| **加购后1小时内购买** | **77.9%** |
| **加购后中位数决策时间** | **7分钟** |

> **结论**：用户不是“遗忘”问题，而是“当场决策”问题。加购后2小时推送优惠券的策略无效。

### 发现二：用户流失的本质是“长时间不来”

基于随机森林流失预测模型（AUC=0.999）：

| 特征 | 重要性 |
|------|--------|
| **最近购买天数 (R_days)** | **96.9%** |
| 品类丰富度 | 1.5% |
| 购买频次 | 0.7% |
| 其他特征合计 | <1% |

> **结论**：流失的最强预测因子是“最近购买天数”，其他因素几乎不产生影响。

### 发现三：RFM 用户分层

基于真实消费金额的RFM分层（90.9万购买用户）：

| 层级 | 人数 | 占比 | 运营策略 |
|------|------|------|----------|
| 高价值VIP | 161,627 | 17.8% | 专属权益、会员日 |
| 核心活跃 | 53,035 | 5.8% | 积分加速 |
| 普通用户 | 226,457 | 24.9% | 常规运营 |
| 新客潜力 | 108,355 | 11.9% | 二次转化引导 |
| 即将流失 | 184,645 | 20.3% | 优惠券召回 |
| 沉睡流失 | 175,091 | 19.3% | 大促唤醒 |

## 💡 业务建议

### 建议一：优化支付链路
- **发现问题**：77.9%用户在加购后1小时内购买，中位数仅7分钟
- **建议方案**：简化加购→支付流程，减少跳转，支持一键下单
- **预期效果**：转化率提升5-8%

### 建议二：流失用户唤醒
- **发现问题**：最近购买天数是流失的最强预测因子
- **建议方案**：对超过15天未访问用户推送唤醒优惠券
- **预期效果**：召回10-15%的流失用户

### 建议三：AB测试验证方案

| 项目 | 内容 |
|------|------|
| 实验假设 | 优化支付链路可提升转化率 |
| 实验组 | 一键下单流程 |
| 对照组 | 现有支付流程 |
| 样本量 | 每组3500个加购用户 |
| 成功指标 | 加购后1小时购买转化率 |
| 实验周期 | 7-14天 |
