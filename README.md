# 电商用户行为分析项目

> 基于 2200万用户 · 2.25亿条真实电商数据的用户增长分析项目

## 📌 项目背景

本项目基于 **Synerise 公开数据集**（2025年发布），对某电商平台2200万用户的2.25亿条行为记录进行全面分析。

**核心问题**：
1. 为什么加购后转化率只有39%？如何提升？
2. 哪些用户最可能流失？如何精准召回？

## 📊 数据集

| 指标 | 数值 |
|------|------|
| 用户数 | 22,000,000 |
| 行为记录 | 2.25亿条 |
| 时间跨度 | 6个月 |

> 数据来源：[Synerise Dataset](https://recsys.synerise.com/data-set)

## 核心分析模块

- **转化漏斗分析**：定位用户流失关键节点。
- **RFM 用户分层**：实现精细化运营。
- **用户决策时间分析**：发现 77.9% 用户在1小时内购买，提出优化支付链路策略。
- **流失预测模型**：使用随机森林识别流失高风险用户。

## 🔍 核心发现

### 发现一：加购后决策极快

| 指标 | 数值 |
|------|------|
| 加购后1小时内购买 | **77.9%** |
| 加购后24小时内购买 | **89.9%** |
| 加购后24小时后购买 | **10.1%** |
| 中位数决策时间 | **7分钟** |

> **结论**：用户不是“遗忘”问题，是“当场决策”问题。加购后2小时推送优惠券已来不及。

### 发现二：流失本质是“长时间不来”

基于随机森林流失预测模型：

| 特征 | 重要性 |
|------|--------|
| 最近购买天数 | **97%** |
| 其他特征合计 | 3% |

> **结论**：用户流失的核心原因是“长时间未访问”，而非价格、品类等因素。

### 发现三：RFM用户分层

| 层级 | 占比 | 策略 |
|------|------|------|
| 高价值VIP | 17.8% | 专属权益 |
| 核心活跃 | 5.8% | 积分加速 |
| 即将流失 | 20.3% | 优惠券召回 |
| 沉睡流失 | 19.3% | 大促唤醒 |

## 💡 业务建议

1. **优化支付链路**：简化加购→支付流程，减少跳转
2. **流失用户唤醒**：对15天未访问用户推送优惠券
3. **AB测试验证**：每组3500样本，7-14天周期
